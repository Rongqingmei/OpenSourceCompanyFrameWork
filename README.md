# OpenSourceCompanyFrameWork
# 开源公司框架
# 简介
软件可以开源，而且迸发了强大的生命力。  
而现在普遍的公司架构，既大股东-CEO-Manager-员工的金字塔架构，100多年了都没有过变化。  
这个公司架构暴露出太多的问题，甚至造成了很多例如贫富差距持续扩大的社会性的问题。  
这不符合上帝给我们指示的平等博爱精神。
理论上公司架构如果开源，让所有员工最低的门槛拥有机会创造价值并拥有价值，将有机会迸发出强大的能量，这股能量甚至可能改变世界。也更符合上帝对我们的指引。  

# 当前社会的公司架构存在的问题
1.公司创造的绝大部分收益，都被股东拿走，绝大部分创造价值的员工只能拿去小部分收益，导致绝大部分人的创造力被全面压制。我不是说股东们就是错误的，股东有钱人也付出投资成本和投资风险，但是这种大部分人创造价值却拿小收益的架构，绝对是全面压制所有人的创造力的。  
  
2.在第一点的前提下，当前的公司架构，获取股份成为股东，也就是创业的过程，对于绝大部分普通人，是一件风险极大的事情，导致创业成为了少数有钱人和少数金字塔顶端的精英的游戏，得看有钱人投资人的眼色。这还是导致了绝大部分人的创造力被全面压制。  
  
3.公司的方向，经常被股份大但是基本什么都不懂的有钱人(很多时候，他们可能是其他行业挣得钱买的股份，对当前行业一窍不通，不少是上一辈有钱，甚至不少是非法所得的钱买下的股份)，和所谓的资历大的CEO把控(甚至连续干倒闭了好多公司的CEO)，大部分真正懂业务，懂行业走向，有经验的员工们，都只能看着列车驶向悬崖，毫无办法，最终还要承担上层的错误后果，被裁员。这也是造成现在公司混乱迷茫和破坏价值创造的重要原因。  
  
4.当前的金字塔型的权力架构，一定会造成拍马屁，欺骗，拉帮结派的价值争夺者上位。而创造价值的，付出脑力，体力的人，被边缘化。因为你的权力来源是你的上级，你只要讨好你的上级，为你的上级提供政治安全(办公室政治)，然后不要暴露出问题给最上面的啥都不懂的股东看出来，你就能一直坐稳和一直上升。而你如果要创造价值，要想改进流程，就会暴露问题，就会威胁到权力来源，让权力来源不开心，你就会被边缘化。这种当前的公司的权力架构，也是一个压制创造力，导致很多成熟公司壮大后开始发疯的一个重要原因。  
  
5.等级跃迁困难，不同等级岗位的学习曲线过于陡峭。一个手机组装的工人，就算他用超过乔布斯300%的智慧和时间花费在他的工作上，他也大概率没法让手机行业有大改进。因为他只能干组装的活，而不是工艺设计。设计和工艺是产品的灵魂，而组装只是最底层最简单的任务。很有可能这位精通了生产流水线的天才，如果能经过学习去参与制造工艺的设计，就能创造巨大的价值。但在现在的公司架构中，这位超过乔布斯300%的天才，只要在这个公司里老实工作，这辈子只能当一名组装工人。学习曲线的陡峭，等级跃迁的困难，导致大部分人的创造潜力被扼杀。

6.公司内部人员的内斗互相竞争消耗严重，选出优秀创造价值的人才效率底下，往往选出的都是内斗争夺价值的强者，而不是面对消费者能创造价值的强者。  

7.在城市与区域的范畴，工作场所过度集中在房价高企的地方，造成普通员工负担过高的房价。老板和股东不在乎高昂的房价，因为他们有钱，他们会把工作场所开在自己住的地方附近，然而普通员工却因此需要负担高昂的房价以及生活成本。
  
# 开源公司框架的愿景
1.股东和员工都能拿到更合理收益分配。激发更强大的创造力。  
  
2.人人皆可参与的，最低的参与创业并分享股权红利的门槛。  
  
3.用开源的机制让真正有业务能力和前瞻力的员工掌控发展方向。  
  
4.普通员工能根据公平开源的价值创造规则升迁。  
  
5.平滑的学习和等级跃迁曲线。  
  
6.想要创造价值的人就能创造价值，凡创造了价值的人，都能获取其收益。

7.把非必须现场工作的员工，工作地点自由分散到生活低的地方。

# 如何实现
1.用去中心化的软件与规则，限制好股东分红与员工收入的比例(一般情况可以直接设置为1:1)，股东无法通过各种技巧来攫取大部分的营收。用来达到股东和员工都能拿到更合理收益分配。  
  
2.用去中心化的，类似于英雄联盟的Rank机制，来进行推进创业的工作和进度，并通过Rank机制分配股权币(创业完成之后由去中心化软件规则，兑现股权)。用来达到人人皆可参与的，最低的参与创业并分享股权红利的门槛。  
  
3.用去中心化的软件与规则，与Rank机制，让时刻变动的，分数高的真正创造价值的人，分配权限与掌控方向。用来达到真正有业务能力和前瞻力的员工掌控发展方向。   
  
4.用去中心化的软件与规则，与Rank机制，创造价值的员工rank分不断上升，通过去中心化的规则，赋予其更多公司权力。  
  
5.用去中心化的软件与规则，让低阶工种达到分段后自动进阶更高阶工种(类似于组装线是白银段位，工艺设计岗位是黄金段位，组装线干熟练了，搭配工艺设计知识的学习，rank分就会上去了，自动分配到工艺设计岗去，类似于白银进阶黄金段位)，由于rank分是动态的，每时每刻一定都会有rank分下降掉段和rank分上升升段的人，用来达到平滑的学习曲线，与必定发生的等级跃迁。  

6.用去中心化的rank机制，让员工们能紧密的与队友团结合作，而不是互相内斗和制约，导致创造效率低下。  
  
7.用开源的机制，让员工能保持高效工作的情况下，把非必须现场工作的员工，工作地点自由分散到生活低的地方。  
  
# 框架实现阶段
1.开源公司框架0.1版本阶段。此阶段是不用技术来实现的纯业务模拟阶段，需要有人来执行和模拟开源公司框架的所有规则，目前这个人主要是我，以及其他有相同愿景的人。大概花费1-3年，建立一个小型的可以盈利和正常运转的小型公司。
  
2.开源公司框架0.2版本阶段。此阶段是用中心化的服务器代码，来执行开源公司框架的所有规则。大概花费是1年，把人工执行的规则，用技术手段变成中心化的服务器代码执行。
  
3.开源公司框架1.0版本阶段。此阶段是用去中心化的DAPP来生效所有开源框架规则。大概花费是2年，把中心化的服务器代码的规则，用技术手段变成去中心化的DAPP来执行。
  

# 开源公司生命周期
1.筹备期。这个时期，发起人将会商定好创业公司的目标(例如马斯克的把人类变成多行星物种)，以及公司的变现规模(例如每年营收100w)。  
  
2.创业期。这个时期，所有人都可以参与创业过程并通过rank机制赚取股权币(允许交易)，所有的纯利润都将按股权币占比进行分配。  
  
3.成熟期。一旦创业期的营收目标，达到了变现规模，将举行全体投票，通过后所有股权币变现为真正公司的股权，公司进入成熟期，不再产生新的股权币。纯利润按规则比例(默认1:1)分配给股东和员工收入。员工的收入综合根据工龄和rank分段来进行分配。  
  
筹备期变现规模的数值很重要，如果这个数字设定的过小，等营收到达之后，由于公司成熟后不再产生股权比，后续的人会更少的倾向于继续加入。如果这个数字设定的过大，某些早期投入大产出小，后期投入小产出大的创业项目，会因为迟迟达不到变现目标，会被后来者轻而易举的稀释股权。  

# 开源公司的三大角色
## 角色分类
参与开源公司的员工，分为三大角色：战略家，战术家，战士。所有员工都能同时是某一种，或者多种角色。  
  
战略家：决定公司的方向和战略决策。  
战术家：决定某个方向的具体实施方案，采用什么业务方法，采用什么技术，等等。  
战士：实现业务，实现技术，推进战术的实现。  
  
## 关于rank
这三个大角色每个角色都有一个rank池，如果你选择了当战略家，就需要参与战略家的rank排名，并获得你的排名和段位。  
  
## 关于分权与制衡
战士的rank打分机制，由战士自评+战术家的评价决定。战术家的rank打分机制，由战术家自评+战略家的评价决定。战略家的rank分，由战略家日常对市场分析的预测考试，以及所有人评分来决定(例如预测某个新出的游戏，下个月的流水，然后下个月所有人需要打分来判定是否预测符合现实)。  
既：  
  
现实世界+战术家+战士，制约战略家。  
战略家，制约战术家。  
战术家，制约战士。  

## 角色的工作时期
### 战略家
1.搜题期。(搜集市场分析的考题)  
2.选题期。(选择哪些考题)  
3.做题期。(参与战略家rank的人员进行做题)  
4.投票期。(公司所有员工投票)  
5.工作期间。(配置人员比例等)  
6.工作成果战士期间。  
7.rank投票期间。  

### 战术家
1.工作期间。  
2.工作成果展示期间。  
3.投票期间。  
  
### 战士
1.工作期间。  
2.工作成果展示期间。  
3.投票期间。  
  
# 开源公司的股权币
公司的创业期，按战略家，战术家，战士三大角色的rank排名，进行产生股权币。  

# 开源公司的职业分配
由战略家分配，动态分配公司的不同职业比例(例如会计人员，程序员人员，销售人员比例为1:2:2)。  
以及有哪些任务类型，任务类型对应的rank小组里面的职业分配比例。  

# Rank机制
每次rank比赛的时候，随机组合接近的段位，总rank分也接近的小组。  
按照每个任务，都分配2个随机对照的rank组来推进，这两个组会有一组获胜，最终采用赢的这组的实现方案。  
并且赢的组集体rank排名上升，输的组集体下降。  
这样的机制，将形成良性的竞争，导致你必须和你的队友最紧密的团队合作，也不容易形成几拨固定的利益团体互相内斗。
